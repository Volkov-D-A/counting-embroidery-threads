# Спецификация приложения

## Цель

Приложение импортирует TXT-отчеты расхода нитей, показывает итоговый список DMC
на экране и позже сможет выгружать результат во внешний файл.

Основной пользовательский сценарий:

1. Пользователь открывает TXT-файл отчета.
2. Приложение разбирает блок расхода нитей.
3. Приложение объединяет одинаковые коды DMC и раскладывает смеси.
4. Пользователь видит итоговую таблицу с палитрой, метражом и количеством мотков.
5. Пользователь может исправить код DMC в итоговой таблице и пересчитать результат.

## Технологии

- Desktop framework: Wails v2.12.0.
- Backend: Go.
- Frontend: React + TypeScript.
- UI language: русский.
- Palette source: открытый датасет `sharlagelfand/dmc`, дополнения DMC 01-35 и DMC Light Effects.

## Входные файлы

Исходные примеры лежат в `examle/`.

Поддерживаемый формат:

- TXT.
- Колонки разделены табами.
- Возможные кодировки: Windows-1251 и UTF-8/ASCII.
- Возможные окончания строк: LF, CRLF, CRCRLF.
- Возможные заголовки:
  - `Длины нитей`
  - `Длины нитей для ...`
  - `Thread lengths`
- Возможные заголовки таблицы:
  - `Нить (DMC)`, `Описание`, `Стежки`, `Бэкстичи`, `Длина (м)`, `Бэкстичи`, `Всего`
  - `Thread (DMC)`, `Description`, `Stitches`, `Backstitches`, `Length(m)`, `Backs`, `Total`

Блок `Bead Count` игнорируется полностью.

## Используемые поля

Для расчета используются:

- код нити из колонки `Нить (DMC)` / `Thread (DMC)`;
- общий метраж из колонки `Всего` / `Total`.

Колонки количества стежков и бэкстича сохраняются только как диагностические данные
для будущей проверки импорта. Метраж не пересчитывается по стежкам.

## Нормализация кодов

Правила нормализации:

- убрать ведущие и конечные пробелы;
- убрать пробелы внутри кода;
- привести буквы к верхнему регистру;
- заменить кириллические буквы, похожие на латиницу, например `Е` -> `E`, `К` -> `K`, `С` -> `C`;
- для компонента смеси `5200` использовать `B5200`, если в палитре нет `5200`;
- для кодов вида `E321(K29)` использовать сам код в таблице, но цветовой свотч сначала искать по металлизированному коду `E321`;
- если металлизированного кода нет в палитре, искать цветовой свотч по базовому DMC-коду, например `E797(K34)` -> `797`;
- если точного буквенного кода нет в палитре, но внутри есть базовый номер DMC, использовать свотч базового номера, например `C797`, `S797`, `D797` -> `797`;
- для кодов `1-9` использовать форму `01-09`, если это новые цвета DMC 01-35.

Если цвет не найден в палитре, приложение показывает серый свотч и предупреждение.
В статусе показываются только читаемые замены кода, например `5200 -> B5200` или `ECRUT -> ECRU`.
Технические замены похожих кириллических букв на латинские в статус не выводятся.

## Расчет расхода

Обычная строка:

```text
итог[код] += Total
```

Смесь из двух цветов:

```text
A+B с Total = 10 м
A += 5 м
B += 5 м
```

Смесь из трех и более цветов:

```text
A+B+C с Total = 10 м
A += 5 м
B += 5 м
C += 5 м
```

Важно: для смесей из 3+ компонентов метраж каждого компонента все равно равен
половине `Total`, а не `Total / количество компонентов`.

Дубликаты одинаковых нормализованных кодов объединяются.

Бэкстич включен в общий расход, потому что используется колонка `Всего`.

Запас не добавляется: он уже заложен на этапе экспорта.

## Корректировки кодов

После импорта пользователь может изменить код DMC в итоговой таблице.

При корректировке:

- новый код проходит те же правила нормализации, что и код из файла;
- метраж исправленной строки переносится на новый код;
- если новый код уже есть в результате, метражи объединяются;
- палитра и количество мотков пересчитываются заново;
- общий метраж не меняется;
- общее количество мотков может измениться из-за объединения строк и округления вверх;
- в статусе строки показывается пометка `исправлено: старый код -> новый код`.

## Мотки

Длина мотка по умолчанию: `8 м`.

Пользователь может изменить длину мотка.

Количество мотков считается с округлением вверх:

```text
мотки = ceil(метраж / длина_мотка)
```

## Интерфейс MVP

Первый экран приложения:

- кнопка выбора TXT-файла;
- числовое поле `Длина мотка, м`;
- компактная сводка в верхней панели вместо заголовка:
  - количество цветов;
  - количество строк импорта;
  - количество цветов без палитры;
- таблица результата:
  - цветовой свотч;
  - редактируемый код DMC;
  - название цвета из встроенной палитры;
  - метраж;
  - мотки;
  - статус палитры/нормализации.

Если код не найден в палитре, в колонке названия показывается `не найдено`.
Если для строки нет пометок палитры, нормализации или исправления, в статусе показывается `ОК`.

Кнопка `Палитра` открывает модальное окно просмотра встроенной палитры DMC.
В окне показываются свотч, код, название и HEX, доступен поиск по всем этим полям.
Там же можно добавить или изменить пользовательский цвет. Пользовательская запись с тем же кодом перекрывает встроенный цвет.
Пользовательская палитра хранится рядом с исполняемым файлом приложения в `user_palette.json`.

Кнопка `Настройки` открывает правила преобразования кодов по колонке `Description`.
Правило задает столбец условия (`1-й: код` или `2-й: описание`), режим совпадения (`равно`, `содержит`, `начинается`, `заканчивается`), значение для проверки, удаляемый префикс текущего кода, добавляемый префикс и суффикс.
Правила применяются по порядку: результат одного правила передается следующему.
Например правило `Description = DMC:E`, `Префикс = E` преобразует код `321` в `E321`.
Например правило по первому столбцу `начинается DMC`, `Убрать = DMC`, а затем правило по описанию `Etoile:C`, `Префикс = C` преобразуют `DMC738` в `C738`.
Настройки хранятся рядом с исполняемым файлом приложения в `transform_settings.json`.

Плавающие кнопки `RU` и `ENG` открывают модальное окно с компактным результатом:

```text
310   1шт. (0,4м)
341   1шт. (1,65м)
```

Для `ENG` используются единицы `pcs.` и `m`. В модальном окне есть кнопка копирования всего текста в буфер обмена.

## Экспорт

Экспорт результата во внешний файл нужен, но формат будет выбран позже.

## Проверки

Минимальный набор тестов:

- импорт всех файлов из `examle/`;
- Windows-1251 и ASCII/UTF-8;
- нормализация кириллических букв, похожих на латиницу, например `Е` -> `E`, `К` -> `K`, `С` -> `C`;
- `5200` -> `B5200` в смесях;
- объединение дублей;
- смеси из 2 цветов;
- смеси из 3+ цветов;
- игнорирование `Bead Count`;
- расчет мотков с округлением вверх.
- корректировка кода с объединением строк и повторным расчетом мотков.
